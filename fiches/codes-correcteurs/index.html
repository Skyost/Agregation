Codes correcteurs d’erreurs

Codes correcteurs d’erreurs

algebra

Petite fiche résumant ce qu’il faut savoir sur les codes correcteurs d’erreurs pour l’agrégation.

Pour tout le document, on fixe p un nombre premier, k et n deux entiers non nuls et q=pk.

Théorie générale

Il s’agit, dans un premier temps, de choisir le corps Fq​ en fonction de l’information que l’on cherche à coder. Par exemple, le choix de F2​ semble le plus naturel pour représenter la manière dont est stockée l’information dans un ordinateur, tandis que F4​ serait plus approprié vis-à-vis de l’ADN.

Définition 1. On appelle mot un vecteur à coefficients dans Fq​.

Après avoir choisi Fq​ comme alphabet, il reste à choisir l’ensemble des mots C du code. Plus précisément :

Définition 2. On appelle code correcteur (ou simplement code) de taille n un sous-ensemble de Fqn​.

Remarque 3. Le code correcteur C est l’ensemble des mots que l’on est en mesure de produire par codage : il ne peut pas occuper l’espace Fqn​ entier, sinon tous les mots seraient valides !

Si l’on reçoit un mot qui n’est pas dans le code, on est donc sûr qu’il y a eu une erreur de transmission. L’opération de codage ajoute une information pour distinguer les mots valides des autres. C’est uniquement lors du décodage que l’on va pouvoir réparer une ou plusieurs erreurs. Le procédé général étant le suivant :

  1. on transforme un message m en un mot c du code (c’est le processus de codage) ;

  2. pendant la transmission, c est altéré en c′ (c’est le processus de transmission) ;

  3. on essaye de déterminer si c′ est un mot du code (c’est le processus de détection d’erreur) ;

  4. on essaye de retrouver c à partir de c′ (c’est le processus de correction d’erreur) ;

  5. on retrouve le message m à partir de c (c’est le processus de décodage).

Exemple 4 (Bit de parité). Dans un ordinateur, chaque mot est coupé en sous-mots de 7 bits, c’est-à-dire, en vecteurs formés de 7 éléments de F2​. Lors du codage de chaque vecteur, on ajoute un bit dit de parité.

Ainsi, soit b1​,…,b7​ une suite de 7 bits. On calcule : b8​=b1​+⋯+b7​mod2 Si le nombre de bits égaux à 1 est pair, b8​=0, sinon, b8​=1. Ainsi, le mot (b1​,…,b8​) a toujours un nombre de bits égaux à 1 qui est pair. On peut alors détecter, à la lecteur d’un mot, si une erreur a eu lieu lors de sa réception : il y aura un nombre impair de bits égaux à 1.

Dans le cadre d’un mot de taille 2, on peut représenter la situation par un cube. Sur cette illustration, nous voyons en turquoise l’ensemble des mots C du code. Une unique erreur correspond à un déplacement sur le cube le long d’une arête. Dans ce cas, le récepteur reçoit un point noir dont la somme de toutes les lettres est un entier impair. En revanche, un tel point est toujours à proximité de trois points turquoise, le récepteur ne dispose donc d’aucun moyen pour une correction automatique.

Ainsi, ce code, a deux inconvénients :

  • il est impossible de détecter où l’erreur a eu lieu, et donc, de la corriger ;

  • si deux erreurs ont lieu, il est impossible de les détecter (car alors, le nombre de bits égaux à 1 reste pair).

L’exemple précédent montre bien qu’il est nécessaire de pouvoir évaluer les propriétés qualitatives d’un code. Ainsi :

Définition 5. Soient x et y deux mots de Fq​ de taille n.

  • Le poids de x, noté ω(x), est le nombre de coefficients non nuls dans x.

  • La distance de Hamming entre x et y, notée dH​(x,y) est définie par dH​(x,y)=ω(x−y)

Proposition 6.

  1. dH​ correspond aux nombres de coefficients qui diffèrent entre x et y.

  2. dH​ est une distance sur Fqn​.

Démonstration.

  1. Soient x,y∈Fqn​. Par définition, ω(x−y) est égal au nombre de coefficients non nuls de x−y, soit au nombre de coefficients qui diffèrent entre x et y.

  2. Soient x,y,z∈Fqn​.

    1. On a dH​(x,y)≥0 par positivité de ω et dH​(x,y)=0 si et seulement s’il y a 0 coefficients qui diffèrent entre x et y ie. x=y.

    2. Le nombre de coefficients non nuls de x−y est égal au nombre de coefficients non nuls de y−x. Donc, dH​(x,y)=ω(x−y)=ω(y−x)=dH​(y,x)

    3. On note (xi​)i∈[[1,n]]​, (yi​)i∈[[1,n]]​ et (zi​)i∈[[1,n]]​ les coefficients respectifs de x, y et z. Soient A={k∈[[1,n]]∣xk​=yk​}, B={k∈[[1,n]]∣yk​=zk​} et C={k∈[[1,n]]∣xk​=zk​}. On a, (A∩B)⊆C En passant au complémentaire, (cC⊆(c(A∩B)=(cA∪(cB D’où, ∣(cC∣d(x,z)​​≤∣(cA∣d(x,y)​​+∣(cB∣d(y,z)​​

 ◻

La distance dH​ permet de quantifier la notion de mot le plus proche. Avec elle, on peut donner la définition suivante.

Définition 7. Soit C un code. On appelle distance minimale de C, l’entier suivant : x,y∈Cmin​{dH​(x,y)∣x=y}

Plus la distance minimale d’un code est grande, plus les mots vont être espacés les uns des autres. En ne prenant en compte que la plus petite des distances, on va pouvoir s’assurer que le code est en mesure de corriger une erreur sous certaines conditions.

Définition 8. Un code C est dit t-correcteur s’il peut corriger au maximum t erreurs.

Remarque 9. Cela signifie que, si x∈C désigne un mot codé et x′∈Fqn​ le mot réceptionné, alors on est en mesure de retrouver le mot x original si d(x,x′)≤t.

Proposition 10. Soit C un code de distance minimale d. On suppose d≥2t+1. Alors, C est t-correcteur.

Démonstration. Soient x,y∈C deux mots distincts du code. Alors, dH​(x,y)≥2t+1 les boules B(x,t) et B(y,t) sont disjointes. Ainsi, soient a∈C un mot codé émis et a′∈Fqn​ le mot réceptionné. Si d(a,a′)≤t, alors a′∈B(a,t) et n’appartient pas à une autre boule de centre un mot du code et de rayon inférieur ou égal à t : on peut corriger a′. ◻

Remarque 11. Notons qu’alors t≤2d−1​⟹t=⌊2d−1​⌋

Codes linéaires

Nous allons maintenant observer ce qui se passe en imposant une structure sur le code.

Définition 12. Un code linéaire C de taille n et de dimension m sur Fq​ est un sous-espace vectoriel de dimension m de Fqn​.

Soit alors une base de C. On considère G une matrice dont les colonnes sont les vecteurs de cette base. On dit que G est une matrice génératrice de C.

Proposition 13. Soit C un code linéaire de taille n et de dimension m sur Fq​. Soit G une matrice génératrice de C. On a, C={Gx∣x∈Fqn​}

Démonstration. Soit (v1​,…,vm​) une base de C. On considère la matrice génératrice de C associée, que l’on note G.

Alors, ∀i∈[[1,m]], en notant ei​ le i-ième vecteur de la base canonique de Fqn​, on a Mei​=vi​. Donc, par linéarité, {Gx∣x∈Fqn​}⊆C. Et comme vi​=Mei​, on a bien l’inclusion réciproque. ◻

Remarque 14. Dans le cadre d’un code linéaire C, la distance minimale d s’exprime alors d=x,y∈Cmin​{dH​(x,y)∣x=y}=x∈Cmin​{ω(x)∣x=0}

Proposition 15. Soit C un code linéaire de taille n et de dimension m sur Fq​. Il existe une matrice H∈Mn−m,n​(Fq​) telle que ∀x∈Fqn​,x∈C⟺Hx=0

Démonstration. On considère le produit scalaire canonique sur Fqn​ : ⟨.,.⟩:((x1​,…,xn​),(y1​,…,yn​))↦i=1∑n​xi​yi​ et C⊥ l’orthogonal de C pour ce produit scalaire. C⊥ est un sous-espace vectoriel de Fqn​ de dimension n−m, dont on note (v1​,…,vn−m​) une base. Définissons H comme étant la matrice dont la i-ième ligne est vi​ pour tout i∈[[1,n−m]]. Soit x∈Fqn​. Alors, on a Hx=0​⟺∀i∈[[1,n−m]],⟨vi​,x⟩=0⟺x∈(C⊥)⊥⟺x∈C​ On aurait aussi pu se contenter de considérer le noyau à gauche de la matrice génératrice (c’est une caractérisation plus commode à implémenter en algorithmique). ◻

Définition 16. En reprenant les notations précédentes, H est appelée matrice de contrôle du code C.

Il s’agit là d’un critère extrêmement pratique pour permettre de tester l’appartenance d’un mot au code.

Exemple 17 (Code de répétition). On se place sur le corps F2​. L’idée est d’envoyer plusieurs copies de chaque bit à être transmis. Ainsi, sur F24​, le code C est composé de deux mots : (0,0,0,0) et (1,1,1,1) Des matrices génératrices G et de contrôle H sont données par G=​1111​​ et H=​111​100​010​001​​ On corrige en remplaçant un message reçu reconnu erroné par le message émis potentiel le plus proche (c’est-à-dire avec le moins de bits différents). Par conséquent, le codage par répétition permet de corriger correctement une erreur portant sur un seul bit mais ne permet pas de corriger correctement une erreur portant sur deux bits.

Proposition 18 (Borne de Singleton). Soit C un code linéaire de taille n, de dimension m et de distance minimale d sur Fq​. Alors, d≤n−m+1

Démonstration. Pour prouver ceci, exhibons un mot x de C de poids inférieur ou égal à n−m+1 (car alors, on aura d≤ω(x)≤n−m+1). Soit F, le sous-espace vectoriel de Fqn​ constitué des vecteurs dont les m−1 dernières coordonnées sont nulles. C’est un espace de dimension n−m+1, et la formule de Grassmann donne : dim(C∩F)​=dim(C)+dim(F)−dim(C+F)=m+n−m+1−dim(C+F)=n+1−dim(C+F)≥n+1−n=1​ Il existe donc x=0 dans C∩F, et ce mot a un poids inférieur ou égal à n−m+1. ◻

Ce dernier résultat illustre le choix à faire entre capacité de correction, et redondance de l’information transmise.

Terminons cette sous-section par la méthode pratique permettant de corriger un mot reçu. Pour cela, on a besoin d’une dernière définition.

Définition 19. Soit C un code linéaire de taille n et de dimension m sur Fq​. Soit H une matrice de contrôle de C. On appelle syndrome d’un mot x∈Fqm​ le vecteur Hx.

Imaginons maintenant que l’on réceptionne un mot a′∈Fqm​. On calcule son syndrome via une matrice de contrôle H et on a deux cas :

  • Le syndrome est nul : a′∈C : on considère alors qu’il n’y a pas d’erreur.

  • Le syndrome est non nul : il existe a∈C (le mot d’origine) et e∈Fqn​ (l’erreur) tels que a′=a+e. Alors, Ha′=H(a+e)=Ha+He=He En notant hj​ le j-ième vecteur colonne de H et ej​ la j-ième coordonnée de e : Ha′=j tel que ej​=0∑​hj​ej​(∗) On en déduit e en résolvant le système (∗). Il est possible que ce système n’ait pas de solution, s’il y a trop d’erreurs par exemple. S’il y a une solution, elle est unique et on peut effectuer la correction : a=a′−e.

Codes cycliques

Nous avons vu dans la section précédente qu’imposer une structure d’espace vectoriel sur un code rendait le codage de l’information beaucoup plus simple via les matrices génératrices. Renforçons davantage la structure de notre code et observons les conséquences.

Définition 20. Soit C un code linéaire de taille n et de dimension m sur Fq​. C est dit cyclique s’il est stable par décalage circulaire, ie. (a0​,a1​,…,am−1​)∈C⟹(a1​,…,am−1​,a0​)∈C

Notons maintenant φ:Fqn​(a0​,…,an−1​)​→↦​Fq​[X]/(Xn−1)∑i=0n−1​ai​Xi​​

Lemme 21. φ est un isomorphisme d’espaces vectoriels.

Démonstration. On sait (par la théorie des corps), que Fq​[X]/(Xn−1) est un espace vectoriel sur Fq​ de dimension n. En effet, en notant X la classe de X dans Fq​[X]/(Xn−1) :

  • La famille (1,…,Xn−1) est libre. Soient λ0​,…,λn−1​∈K tels que i=0∑n−1​λi​Xi=i=0∑n−1​λi​Xi​=0 Alors, le polynôme ∑i=0n−1​λi​Xi est dans l’idéal (Xn−1), mais est de degré strictement inférieur à n. Donc ses coefficients sont nuls : on a ∀i∈[[1,n]], λi​=0.

  • La famille (1,…,Xn−1) est génératrice. Soit P∈Fq​[X]/(Xn−1). On fait la division euclidienne de P par Xn−1 dans Fq​[X] : ∃(Q,R)∈Fq​[X] tel que P=Q(Xn−1)+R avec deg(R)<n ou R=0 En repassant modulo (Xn−1), on a bien P=R de degré inférieur à n, donc appartenant à l’espace vectoriel engendré par (1,…,Xn−1).

Ainsi, Fqn​ et Fq​[X]/(Xn−1) sont isomorphes en tant qu’espaces vectoriels de même dimension sur Fq​. L’application φ étant surjective et linéaire (par définition), on a bien un isomorphisme. ◻

À l’aide de cet isomorphisme, nous allons pouvoir identifier un code linéaire de taille n sur Fq​ à un sous-espace vectoriel C=φ(C) de Fq​[X]/(Xn−1). Ce raisonnement va nous permettre de caractériser les codes cycliques.

Proposition 22. Soit C un code linéaire de taille n. Alors, C est cyclique si et seulement si C=φ(C) est un idéal de Fq​[X]/(Xn−1).

Démonstration. Soient a=(a0​,a1​,…,an−1​)∈C et a′=(an−1​,a0​,…,an−2​). Remarquons que, φ(a′)​=an−1​+i=0∑n−2​ai​Xi+1​=an−1​(1−1+Xn)+i=0∑n−2​ai​Xi+1=i=0∑n−1​ai​Xi+1=Xi=0∑n−1​ai​Xi=Xφ(a)​

  • Supposons C cyclique. Alors, par ce qu’on vient de dire, C est stable par multiplication par X. Mais C est un sous-espace vectoriel de Fq​[X]/(Xn−1), donc il est aussi stable par addition et par multiplication par un scalaire. Finalement, C est bien un idéal de Fq​[X]/(Xn−1).

  • Supposons C idéal de Fq​[X]/(Xn−1). Alors, C est stable par multiplication par X. Donc par le raisonnement précédent, C est clairement cyclique.

 ◻

Nous arrivons au théorème suivant qui nous indique que, pour fabriquer un code cyclique de dimension m, il suffit de savoir factoriser Xm−1 dans Fq​[X] (ce qui peut se faire par l’algorithme de Berlekamp).

Théorème 23 (Structure des codes cycliques). Soit m∈[[0,n]].

  1. Soit P=∑i=0n−m​ai​Xi un diviseur unitaire de Xn−1 dans Fq​[X]. Soit a=φ−1(P) le mot correspondant à P. Alors, en notant σ:Fqn​→Fqn​ l’application de permutation circulaire, C=Vect(σi(a))i∈[[0,m−1]]​(∗) forme un code cyclique de dimension m.

  2. Réciproquement, si C est un code cyclique de dimension m sur Fqn​, il existe un polynôme P∈Fq​[X] diviseur de Xn−1 vérifiant (∗) pour C.

Démonstration.

  1. Clairement, C=Vect(σi(x))i∈[[0,m−1]]​ est un sous-espace vectoriel de Fqn​[X] de dimension m : c’est un code linéaire. Reste à montrer qu’il est cyclique. Soit b=∑i=0m−1​bi​σi(a) un mot de C. Il s’agit de montrer que σ(b)∈C. Or, σ(b)=i=0∑m−1​bi​σi+1(a)=bm−1​σm(a)+i=1∑m−1​bi−1​σi(m) et, d’après la base choisie pour C, ∑i=1m−1​bi−1​σi(m)∈C. Reste à montrer que bm−1​σm(a)∈C. On a, φ(bm−1​σm(a))​=bm−1​Xmφ(a)=bm−1​XmP​ Or, P est de degré n−m, unitaire et divise Xn−1, donc il existe Q∈Fq​[X] unitaire de degré m tel que Xn−1=PQ. D’où, φ(bm−1​σm(a))​=bm−1​(Xm−Q​)P+bm−1​Q​P=bm−1​(Xm−Q​)P​ Comme bm−1​(Xm−Q) est de degré au plus m−1, on peut l’écrire ∑i=0m−1​ci​Xi. Ainsi, φ(bm−1​σm(a))​=i=0∑m−1​ci​XiP=i=0∑m−1​ci​φ(σi(a))=φ(i=0∑m−1​ci​σi(a))∈φ(C)​ D’où bm−1​σm(a)∈C : on a bien ce qu’on voulait.

  2. Soient C un code cyclique de dimension m sur Fqn​ et π=π(Xn−1)​ la projection de Fq​[X] sur le quotient Fq​[X]/(Xn−1). Alors, d’après la Proposition 22, φ(C)=C est un idéal de Fq​[X]/(Xn−1), donc π−1(C) est un idéal de Fq​[X], qui est principal par principalité de Fq​[X]. On peut noter P le générateur unitaire. Montrons que P∣Xn−1.

    C est un idéal de Fq​[X]/(Xn−1), donc 0∈C, donc Xn−1∈π−1(C) : il existe Q∈Fq​[X] tel que Xn−1=QP. On a bien P∣Xn−1.

    Il s’agit maintenant de montrer que P est bien de degré n−m. Notons k=deg(P). Soit E={h∈Fq​[X]∣deg(h)∈[[0,n−k−1} On a, π(P⋅E)={Ph∈Fq​[X]/(Xn−1)∣deg(h)∈[[0,n−k−1} et P⋅E⊆π−1(C)⟹π(P⋅E)⊆C.

    Soit R∈π−1(C). Par définition de P, il existe S∈Fq​[X] tel que R=PS. On effectue la division euclidienne de S par Q : ∃(T,U)∈Fq​[X] tel que S=QT+U avec deg(U)<n−k ou U=0 d’où : R⟹π(R)​=P(QT+U)=T(Xn−1)+PU=π(PU)∈π(P⋅E)​ Ainsi, on a C⊆π(P⋅E). On a alors montré que C=π(P⋅E). Or, ∣π(P⋅E)∣=qn−k et C est un sous-espace vectoriel de Fqn​ de dimension m. Par isomorphisme, on a donc : ∣C∣=qm=∣C∣ ce qui permet de conclure que m=n−k.

    Pour terminer, on écrit P=∑i=0n−m​ai​Xi et on considère a=(a0​,…,am−1​)∈Fqm​. Comme C est un idéal de Fq​[X]/(Xn−1), ∀i∈[[0,m−1]],XiP∈C⟹∀i∈[[0,m−1]],σ(a)i∈C Et (σi(a))i∈[[0,m−1]]​ est une famille libre de cardinal m, donc est bien une base de C.

Étude d’un code de Hamming

D’après Wikipédia, un code de Hamming est un code correcteur linéaire. Il permet la détection et la correction automatique d’une erreur si elle ne porte que sur une lettre du message. Un code de Hamming est parfait : pour une longueur de code donnée il n’existe pas d’autre code plus compact ayant la même capacité de correction. En ce sens son rendement est maximal. Il existe une famille de codes de Hamming ; le plus célèbre et le plus simple après le code de répétition binaire de dimension 3 et de longueur 1 est sans doute le code binaire de longueur 7, de dimension 4 et de distance minimale 3 : ça tombe bien, il est au programme de l’option C de modélisation !

Définition 24. Le code Hamming CH​ de longueur 7 permet de coder un mot de longueur 4 en un mot de code de longueur 7. C’est un code linéaire, dont une matrice génératrice est GH​=​1101000​0110100​0011010​0001101​​∈M7,4​(F2​)

Exemple 25. On souhaite coder le mot (1,0,0,1). On calcule : GH​​1001​​=​1100101​​ Le mot codé est donc (1,1,0,0,1,0,1).

On peut en fait expliciter les mots de ce code : il y en a 24=16.

Proposition 26.

  1. CH​ a une distance minimale de 3.

  2. CH​ est 1-correcteur.

  3. H=​100​010​001​110​011​111​101​​ est une matrice de contrôle de ce code.

Démonstration.

  1. Le minimum des poids est bien 3 d’après le tableau précédent.

  2. D’après la Remarque 11, la capacité de correction du code est égale à ⌊23−1​⌋=1

  3. Soit x∈F27​. On note (v0​,…,v3​) la base de CH​ associée à GH​. Alors, x∈C⊥​⟺∀i∈[[0,3]],⟨x,vi​⟩=0⟺x=ah1​+bh2​+ch3​ pour a,b,c∈F2​​ où h1​=(1,0,0,1,0,1,1), h2​=(0,1,0,1,1,1,0) et h3​=(0,0,1,0,1,1,1). Donc (h1​,h2​,h3​) est une base de C⊥, ce qui mène au résultat voulu.

 ◻

Proposition 27. Le code de Hamming est cyclique, engendré par P=X3+X+1.

Démonstration. Les 4 vecteurs colonnes de la matrice GH​ se déduisent les uns des autres par permutation circulaire. Par conséquent, l’ensemble du code est invariant par permutation circulaire : CH​ est bien cyclique. Et le polynôme P correspond au mot (1,1,0,1,0,0,0) qui est le premier vecteur colonne de la matrice GH​. ◻

En pratique, le code de Hamming se manipule de la manière suivante :

  1. On a un mot x∈F24​. On calcule a=GH​x et on envoie a.

  2. Le receveur reçoit a′. Il calcule le syndrome s=Ha′. Si s est nul, il pose a=a′. Sinon, il pose a=a′+ej​ où pour j∈[[1,7]], s=Hej​.

  3. Le receveur résout GH​x=a.
